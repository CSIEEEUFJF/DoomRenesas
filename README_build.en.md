# Build, Flash, and Debug

## Requirements

- Renesas e2 studio with the Synergy project support installed.
- The GNU Arm Embedded toolchain used by the project build.
- A connected S7G2 Starter Kit for flashing and debug.

## Build flow

The project is built from the generated Synergy and Doom sources as a standard e2 studio project. Use the `Debug` configuration unless you have a specific reason to create another build profile.

Typical workflow:

1. Open the project in e2 studio.
2. Let the generated Synergy files remain in sync with the active configuration.
3. Build the `Debug` target.
4. Flash and launch a debug session on the board.

The build produces `DoomRenesas.elf` and `DoomRenesas.srec` in the `Debug` directory.

## Debug flow

The port is designed to be inspected through `doom_error_msg`. When the game stops early or the screen stays blank, check that symbol in the debugger first. It records stage markers, allocation failures, touch errors, and fault handlers.

Useful places to set breakpoints:

- `main_thread_entry()`
- `doomgeneric_Create()`
- `DG_UpdatePalette()`
- `DG_DrawFrame()`
- `doom_set_error_msg()`

## Notes on generated code

Several files in `src/synergy_gen` are generated by e2 studio and the Synergy configuration. If you intentionally reconfigure the project, regenerate those files as a group so the display, SPI, DTC, and GLCDC settings stay aligned.
